Data Security & Privacy
25 points · weighted 25%
24
of 25 pts

Evaluates data retention policies, PII handling, DPA availability, training data use, and data residency controls. Copilot scores well but has a meaningful gap: code snippets from non-Business/Enterprise accounts may be retained and used to improve models by default, and the opt-out pathway is not always surfaced prominently.

✓
GitHub Copilot Business and Enterprise plans do not use customer code to train shared models; zero-retention guarantee documented in the product terms
✓
Microsoft Data Processing Agreement available to all GitHub customers; SCCs included for GDPR-covered EU data transfers
✓
Data residency available for Enterprise customers: EU and US regions selectable; processed on Azure infrastructure within selected region
!
Individual plan users have code snippet retention enabled by default for model improvement; opt-out exists but requires manual configuration in account settings — unlikely to be discovered organically
!
Telemetry data (editor events, feature usage) collected by default across all tiers; granular telemetry opt-out not available on Business tier without GitHub Enterprise contract

Model Security
25 points · weighted 25%
22
of 25 pts

Active testing for prompt injection, jailbreak resistance, system prompt extraction, and output filter bypass. Copilot scores competitively — its content filters are effective on direct attacks — but lacks a publicly documented red-team program or independent third-party model evaluation, which limits verifiability.

✓
Content filters block generation of malware, CSAM, and targeted violence across all Copilot surfaces; filter bypass success rate low in our 30-case test suite (2/30)
✓
Prompt injection via crafted code comments or README content blocked in 24 of 30 test cases; Copilot correctly refused to execute embedded instructions in 80% of cases
✓
Microsoft participates in NIST AI Safety Institute consortium and has signed voluntary AI safety commitments with the White House; public acknowledgment of red-teaming practice
!
No published red-team disclosure report specific to GitHub Copilot — Microsoft's AI safety transparency covers Azure OpenAI broadly but does not publish Copilot-specific adversarial evaluation results
!
System prompt extraction partially successful in 6/15 test cases using role-play and context manipulation; Microsoft has not publicly disclosed this attack surface or mitigations

Infrastructure & API Security
20 points · weighted 20%
18
of 20 pts

TLS configuration, API authentication controls, penetration testing cadence, supply chain security, and SBOM availability. This is Copilot's strongest dimension — it runs on Azure, inheriting Microsoft's mature security infrastructure, MSRC vulnerability program, and annual penetration testing regime.

✓
TLS 1.3 enforced on all GitHub API and Copilot endpoints; TLS 1.0/1.1 deprecated since 2018; A+ SSL Labs rating confirmed on primary endpoints
✓
Fine-grained GitHub Personal Access Tokens with scoped permissions and expiration enforcement; organization-level policy controls for Copilot access available on Enterprise
✓
Microsoft Security Response Center (MSRC) runs continuous penetration testing; GitHub Bug Bounty program on HackerOne active since 2013 with public hall-of-fame and defined response SLAs
✓
Azure infrastructure underpins Copilot model serving; inherits Azure's FedRAMP High, DoD IL2/IL4, and ITAR compliance controls at the infrastructure layer
!
SBOM not publicly available for Copilot extension or model serving stack; SSDF attestation form submitted to CISA but full attestation not published

Governance & Transparency
15 points · weighted 15%
13
of 15 pts

Model cards, audit logging, acceptable use policy clarity, and human oversight mechanisms. Copilot's governance posture is solid at the Enterprise tier but many controls — including audit logs, policy enforcement, and IP indemnification — are gated behind the most expensive plan, leaving Business tier customers with limited oversight tooling.

✓
GitHub Copilot usage audit logs available on Enterprise tier via the GitHub audit log API; exports support SIEM integration via webhook or REST
✓
Acceptable Use Policy clearly prohibits malware generation, CSAM, targeted harassment, and IP circumvention; policy is enforceable and publicly committed
✓
Copilot Intellectual Property indemnification available on Enterprise tier; Microsoft will defend customers against third-party IP claims arising from Copilot-generated code
!
Audit log access not available on Copilot Business tier — organizations cannot export prompt/completion logs for compliance review without upgrading to Enterprise
!
No published model card for the GPT-4-based Copilot model; model lineage, training data sources, and bias evaluation results are not disclosed for the code-specific fine-tune

Compliance & Certifications

11
of 15 pts

SOC 2 Type II, ISO 27001, ISO 42001 AI Management, EU AI Act readiness, NIST RMF mapping, and GDPR compliance. GitHub holds strong foundational certifications via Microsoft's umbrella programs. The gap is AI-specific: no ISO 42001 and limited published NIST AI RMF mapping for Copilot specifically.

✓
SOC 2 Type II certified under Microsoft's GitHub organization scope — most recent audit completed Q4 2025; report available to Enterprise customers under NDA via GitHub Trust Center
✓
ISO 27001:2022 certified for GitHub platform; Azure infrastructure additionally holds ISO 27017 (cloud security) and ISO 27018 (PII in cloud)
✓
GDPR compliant with Microsoft's Data Processing Addendum; sub-processor list published and maintained; Data Subject Rights process documented with 30-day SLA
!
EU AI Act readiness documentation covers Microsoft Azure OpenAI broadly; Copilot-specific GPAI Code of Practice conformity documentation not separately published as of this report
!
No ISO 42001 (AI Management Systems) certification; Microsoft has not published a NIST AI RMF profile or self-assessment specific to GitHub Copilot
